Dotfiles 管理手册

2026.01.05 Linux

利用 git --bare 库管理家目录下的配置文件，既能保持目录整洁，又能实现跨设备同步。

一、 初始化与基础配置

1. 远端仓库初始化

在 GitHub 或 GitLab 上创建一个名为 dotfiles 的私有仓库，然后在本地执行：

Bash

# 初始化 Bare 仓库到指定目录
git init --bare $HOME/.dotfiles

# 设置临时别名（建议将其写入 .zshrc 或 .bashrc）
alias dotfiles='/usr/bin/git --git-dir=$HOME/.dotfiles/ --work-tree=$HOME'

# 隐藏未追踪的文件（避免 git status 列出整个家目录）
dotfiles config --local status.showUntrackedFiles no

# 关联远程库并推送
dotfiles remote add origin git@github.com:hnsylzg/dotfiles.git
dotfiles push --set-upstream origin main

2. 新机器部署 (Clone)

在新环境下快速恢复配置：

Bash

# 克隆仓库
git clone --bare git@github.com:hnsylzg/dotfiles.git $HOME/.dotfiles

# 定义别名
alias dotfiles='/usr/bin/git --git-dir=$HOME/.dotfiles/ --work-tree=$HOME'

# 签出文件到工作区 ($HOME)
dotfiles checkout

注意：如果 checkout 报错提示文件冲突，请先备份或删除本地已存在的同名配置文件。

二、 分支与同步策略

为了在不同机器或环境下使用不同主题（如 Matugen 动态配色 vs 静态配色），建议采用多分支管理。

1. 分支classic创建与推送

Bash

# 创建并立即切换分支
dotfiles checkout -b classic <commit-id>

# 只创建分支，不切换
dotfiles branch classic <commit-id>

# 第一次推送并建立跟踪关系
dotfiles push -u origin classic

# 以后仅需直接推送
dotfiles push

2. Bare 库分支可见性设置

由于是 Bare 库，有时无法直接看到远程分支。需要显式配置 fetch 规则：

Bash

# 修复远程分支追踪关系
dotfiles config remote.origin.fetch "+refs/heads/*:refs/remotes/origin/*"

# 获取更新
dotfiles fetch --all --prune
dotfiles checkout classic

三、 跨分支逻辑同步

当你在 main 分支改进了功能（如快捷键），希望同步到 classic 分支，但又不想要 main 的配色时，可采用以下方案：

方案 A：精准拣选 (Cherry-pick)

适用于同步某次特定的代码改动。

Bash

dotfiles checkout classic
dotfiles cherry-pick <commit-id>  # 仅将该提交的改动应用到当前分支
dotfiles push

方案 B：覆盖特定文件 (Checkout file)

适用于直接同步某个脚本或配置文件的最新版本。

Bash

dotfiles checkout classic
# 从 main 分支“借用”特定文件覆盖到当前分支
dotfiles checkout main -- .config/hypr/scripts/waybar_logic.sh

dotfiles commit -m "sync: update waybar logic from main"
dotfiles push

四、 进阶：架构层面的解耦 (强烈推荐)

与其事后同步，不如在设计配置时就实现逻辑与样式的解耦。

1. 目录结构规范

将配置文件拆分为“功能逻辑”与“视觉样式”：

  • keybinds.conf: 存放快捷键、窗口规则（各分支通用）。
  • theme.conf: 存放颜色、边框、阴影（各分支独立）。

2. 引用机制

在主配置文件 hyprland.conf 中使用 source 命令：

代码段

# .config/hypr/hyprland.conf
source = ~/.config/hypr/keybinds.conf
source = ~/.config/hypr/theme.conf

3. 同步工作流

当你修改了快捷键逻辑，只需在 classic 分支执行一行命令即可完成完美同步，绝不污染配色：

Bash

dotfiles checkout main -- .config/hypr/keybinds.conf
