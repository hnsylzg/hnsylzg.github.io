Gentoo 如何深度定制内核以获得极致启动速度

2026.01.05 Linux

深度定制内核是 Gentoo 玩家的“成人礼”，也是将开机速度压制在秒级单位的关键。要获得极致的启动速度，核心逻辑是：把内核做小、把驱动内置、把多余的扫描关掉。

以下是深度定制内核以实现极致启动速度的进阶指南。

一、 预备工作：获取硬件指纹

在精简内核前，你必须明确知道哪些驱动是绝对必须的，否则系统将无法引导（通常导致 Kernel Panic）。

Bash

# 安装硬件检测工具
emerge --ask sys-apps/pciutils sys-apps/usbutils
# 查看当前正在使用的模块（如果你正在使用通用内核或 LiveCD 成功引导）
lsmod
# 获取内核配置建议（非常有用）
lsinitrd /boot/initramfs-xxxx.img | grep modules # 查看 initramfs 依赖了哪些模块

二、 极致精简：内核配置策略 (make menuconfig)

进入 /usr/src/linux，开启你的裁剪之旅。

1. 禁用 Initramfs (启动加速的核心)

Initramfs 的存在是为了加载磁盘驱动，但这会多出一道解压和扫描的过程。

  • 做法： 将你的 磁盘控制器驱动（如 NVMe 或 SATA）和 根文件系统驱动（如 Ext4）直接编译进内核（标记为 <*> 而非 <M>）。
  • 配置位置：
    • Device Drivers -> NVME Support -> <*>
    • File systems -> <*> (例如 Ext4)
  • 结果： 启动时内核直接挂载根分区，无需加载 Initramfs，节省 0.5~2 秒。

2. 精简处理器特性

通用内核会包含对几十年前 CPU 的兼容代码。

  • 做法： 开启针对你当前架构的优化。
  • 配置位置：
    • Processor type and features -> Processor family -> 选择你的具体架构 (如 Core i7 或 Zen)。
    • 关闭 Support for 32-bit ELF binaries（如果你不需要运行 32 位老旧软件）。

3. 去除冗余驱动（最耗时的环节）

内核在启动时会扫描各种总线，关掉你没有的硬件可以减少扫描等待。

  • 必关项：
    • Amateur Radio support (业余无线电)
    • ISDN support (老式调制解调器)
    • Accessibility support (辅助功能)
    • Industrial I/O support (工业传感器)
  • 网络： 只保留你的网卡驱动（通常是 Intel e1000e 或 Realtek），关闭所有其他的 Token Ring, FDDI 设备驱动。

4. 优化控制台回显

大量的内核日志（dmesg）输出到屏幕实际上是非常耗时的（特别是串口控制台）。

  • 做法： * Device Drivers -> Graphics support -> Console display driver support -> 关闭多余的字体。
    • 在引导参数中加入 quiet。

三、 编译与静态化内置

1. 编译参数优化

在编译内核时，确保利用所有 CPU 核心。

Bash

make -j$(nproc)

2. 静态内置 vs 模块

  • 内置 (<\*>): 启动时直接运行，速度最快，内核文件略大。
  • 模块 (<M>): 启动后再按需加载。
  • 极致原则： 启动必须的驱动（显卡、键盘、磁盘）全部内置；非必须的（USB 蓝牙、打印机）做成模块或直接剔除。

四、 配合引导器的极致优化

即使内核做到了极致，引导器（GRUB）的等待时间也可能成为瓶颈。

1. 减少 GRUB 等待

编辑 /etc/default/grub:

Plaintext

GRUB_TIMEOUT=0
GRUB_CMDLINE_LINUX_DEFAULT="quiet rootwait nowatchdog"
  • nowatchdog: 禁用软件看门狗，减少启动时的时钟自检。

2. 使用 EFISTUB (跳过 GRUB)

如果你真的追求极致，可以不使用 GRUB，直接让 UEFI 固件加载内核。

Bash

# 使用 efibootmgr 直接向主板注册内核
efibootmgr --create --disk /dev/nvme0n1 --part 1 --loader /vmlinuz-gentoo --label "Gentoo FastBoot" --unicode "root=/dev/nvme0n1p3 rw quiet"

五、 启动速度度量指标

安装完成后，使用 systemd 自带工具分析瓶颈：

Bash

systemd-analyze
# 详细查看哪个服务拖慢了速度
systemd-analyze blame
# 生成精美的启动流程图
systemd-analyze plot > boot.svg

常见瓶颈修复建议：

  • DHCP 等待： 启动慢往往是因为在等网络分配 IP。改为静态 IP 或使用 NetworkManager 的异步启动。
  • 文件系统检查： 确保 /etc/fstab 中的 pass 字段配置正确，避免每次开机都全盘扫描。
