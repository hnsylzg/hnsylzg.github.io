Gentoo 极致快启：一键内核更新脚本

2026.01.05 Linux

既然我们要实现“极致快”，那就不能把时间浪费在手动输入 make、cp 和 efibootmgr 命令上。

下面这个脚本它会自动完成内核编译、固件嵌入检查、自动挂载 EFI 分区以及更新主板启动项的全流程。

创建一个名为 fast_kernel_update.sh 的文件：

Bash

#!/bin/bash

# --- 配置区 ---
KERNEL_VERSION=$(make -s kernelrelease)
EFI_PART="/dev/nvme0n1p1"  # 你的EFI分区设备路径
MOUNT_POINT="/boot/efi"
LOADER_NAME="Gentoo_FastBoot"
# --- --- --- ---

echo "开始为内核 $KERNEL_VERSION 进行极致加速编译..."

# 1. 自动根据逻辑核心数进行编译
echo "正在编译内核..."
make -j$(nproc) || { echo "编译失败"; exit 1; }

# 2. 挂载 EFI 分区
if ! mountpoint -q "$MOUNT_POINT"; then
    mount "$EFI_PART" "$MOUNT_POINT"
fi

# 3. 拷贝内核到 EFI 分区 (不带 initramfs，因为我们已经内建了驱动)
echo "拷贝内核镜像到 EFI 分区..."
cp arch/x86/boot/bzImage "$MOUNT_POINT/vmlinuz-fast"

# 4. 移除旧的 EFI 启动项并创建新的 (实现 EFISTUB 直接引导)
echo "更新 UEFI 引导项..."
# 删除同名的旧启动项
OLD_ID=$(efibootmgr | grep "$LOADER_NAME" | awk '{print $1}' | sed 's/Boot//;s/\*//')
if [ -n "$OLD_ID" ]; then
    efibootmgr -b "$OLD_ID" -B > /dev/null
fi

# 注册新的引导项 (请根据你的根分区 UUID 修改 root= 部分)
# quiet: 安静模式  rootwait: 等待磁盘就绪  rw: 读写挂载
ROOT_UUID=$(blkid -s UUID -o value /dev/nvme0n1p3) # 自动获取根分区UUID
efibootmgr --create --disk "$EFI_PART" --part 1 \
    --loader /vmlinuz-fast --label "$LOADER_NAME" \
    --unicode "root=UUID=$ROOT_UUID rw quiet nowatchdog lpj=$(dmesg | grep lpj | head -n 1 | awk '{print $NF}')"

echo "完成！现在的启动流程：BIOS -> $LOADER_NAME -> Desktop"
echo "建议下次重启后运行 'systemd-analyze' 查看成绩。"

💡 脚本背后的“快”细节

  1. EFISTUB 直接引导：我们跳过了 GRUB。GRUB 扫描磁盘、显示菜单、加载模块至少需要 1.5 到 3 秒。而 EFISTUB 由主板固件直接拉起内核，时间几乎为 0。
  2. Lpj 参数预设：脚本中的 lpj=$(dmesg | ...) 会抓取你当前 CPU 的 loops_per_jiffy。将其传给内核后，内核启动时就不再需要执行“校准延迟循环”这一检测步骤，节省约 100ms。
  3. 内建驱动 (vmlinuz-fast)：因为你已经把显卡、磁盘固件编译进了内核，所以脚本不再调用 genkernel 或 dracut 生成 initramfs。这省去了内存盘解压的时间。

🏎️ 检查你的“战果”

当你使用该脚本重启后，你的系统启动链条会变成这样：

在进入桌面后，请务必执行：

Bash

systemd-analyze plot > fast_boot.svg

使用浏览器打开这个 svg 文件。你会看到一条非常“陡峭”且紧凑的横条图。如果 Kernel 和 Initrd（由于没用 initrd，这部分应该接近0）的总和在 1 秒以内，那么你已经站在了 Gentoo 优化的巅峰。

最后一步建议：

为了保证系统万无一失，千万不要删掉你的旧内核和旧 GRUB 引导。这个脚本创建的是一个新的 EFI 启动条目。如果新内核 Panic 了，你只需要在主板开机时按 F12（或对应的 BIOS 菜单键）选回原来的 GRUB 即可。
