# PEP 8 规范的核心要点精简总结

以下是 PEP 8 规范的核心要点精简总结，涵盖 Python 代码风格的核心规则：

### 📐 **1. 代码布局**

- **缩进**：使用 **4个空格**，禁用 Tab 键（避免不同编辑器显示差异）。
- **行长度**：每行不超过 **79字符**（文档/注释不超过72字符），长表达式用括号或反斜杠换行。
- 空行：
  - 函数/类之间：**两个空行**分隔。
  - 类内方法之间：**一个空行**分隔。

### 🔤 **2. 命名规范**

- **变量/函数**：小写字母 + 下划线（`snake_case`），如 `calculate_area`。
- **类名**：首字母大写驼峰式（`CamelCase`），如 `UserProfile`。
- **常量**：全大写 + 下划线，如 `MAX_SIZE`。
- **私有属性**：双下划线开头（`__private_var`），受保护属性单下划线（`_protected_var`）。

### ⚙️ **3. 空格与符号**

- **运算符**：两侧加空格（`x = y + 1`），但参数默认值不加（`def func(a=1)`）。
- **逗号/冒号**：逗号后加空格（`[1, 2, 3]`），切片冒号两侧不加空格（`list[1:3]`）。
- **括号**：内部紧贴内容不加空格（`func(a, b)`✅，`func( a, b )`❌）。

### 📝 **4. 注释与文档**

- **行内注释**：`#` 后加空格，与代码间隔两空格（`x = x + 1  # 补偿边界值`）。
- **文档字符串**：模块/类/函数用三重引号 `"""` 描述功能及参数。
- **块注释**：每行以 `#` 开头，与代码同级缩进。

### 📦 **5. 导入规范**

- **顺序**：标准库 → 第三方库 → 本地模块，组间用空行分隔。
- **方式**：每行一个导入，避免通配符（`from module import *`❌）。

### ⚡️ **6. 其他关键规则**

- **单例比较**：用 `is`/`is not` 替代 `==`（如 `if x is None`）。
- **魔法数字**：避免直接使用未命名常量（如用 `ADULT_AGE = 18` 替代 `age > 18`）。
- **字符串引号**：单双引号无区别，但需保持一致性（含单引号时用双引号避免转义）。

### 🛠️ **工具推荐**

- **检查工具**：`flake8`（综合检查）、`pylint`（严格检测）。
- **格式化工具**：`Black`（一键格式化）、`autopep8`（自动修复）。

------

> 💡 **核心原则**：代码是写给人看的，其次才是机器执行。遵循 PEP 8 可显著提升可读性、协作效率和维护性。
>  🔍 完整规范详见 [PEP 8 官方文档](https://peps.python.org/pep-0008/)。
